31 /*
32  Overview of the task interface.
33 
34  The task interface is divided into two parts: a set of proxy classes
35  for accessing task information (TaskProxy, OperatorProxy, etc.) and
36  task implementations (subclasses of AbstractTask). Each proxy class
37  knows which AbstractTask it belongs to and uses its methods to retrieve
38  information about the task. RootTask is the AbstractTask that
39  encapsulates the unmodified original task that the planner received
40  as input.
41 
42  Example code for creating a new task object and accessing its operators:
43 
44  TaskProxy task = new TaskProxy(new RootTask());
45  for (OperatorProxy op : task->get_operators())
46  cout << op.get_name() << endl;
47 
48  Since proxy classes only store a reference to the AbstractTask and some
49  indices, they can be copied cheaply and be passed by value instead of
50  by reference.
51 
52  In addition to the lightweight proxy classes, the task interface
53  consists of the State class, which is used to hold state information
54  for TaskProxy tasks. The State class provides methods similar to the
55  proxy classes, but since State objects own the state data they should
56  be passed by reference.
57 
58  For now, only the heuristics work with the TaskProxy classes and hence
59  potentially on a transformed view of the original task. The search
60  algorithms keep working on the original unmodified task using the
61  GlobalState, GlobalOperator etc. classes. We therefore need to do two
62  conversions: converting GlobalStates to State objects for the heuristic
63  computation and converting OperatorProxy objects used by the heuristic
64  to GlobalOperators for reporting preferred operators. These conversions
65  are done by the Heuristic base class. Until all heuristics use the new
66  task interface, heuristics can use Heuristic::convert_global_state() to
67  convert GlobalStates to States. Afterwards, the heuristics are passed a
68  State object directly. To mark operators as preferred, heuristics can
69  use Heuristic::set_preferred() which currently works for both
70  OperatorProxy and GlobalOperator objects.
71 
72  int FantasyHeuristic::compute_heuristic(const GlobalState &global_state) {
73  State state = convert_global_state(global_state);
74  set_preferred(task->get_operators()[42]);
75  int sum = 0;
76  for (FactProxy fact : state)
77  sum += fact.get_value();
78  return sum;
79  }
80 
81  For helper functions that work on task related objects, please see the
82  task_tools.h module.
83 */
