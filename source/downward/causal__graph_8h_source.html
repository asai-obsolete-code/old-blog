21 /*
22  An IntRelation represents a relation on a set {0, ..., K - 1} as an
23  adjacency list, encoded as a vector<vector<int> >. For example, the
24  relation { (0, 1), (0, 3), (1, 3), (3, 0), (3, 1), (3, 2) } over the
25  set {0, 1, 3, 4} would be represented as
26 
27  [
28  [1, 3], # representing (0, 1), (0, 3)
29  [3], # representing (1, 3)
30  [], # there are no pairs of the form (2, v)
31  [0, 1, 2], # representing (3, 0), (3, 1), (3, 2)
32  [] # there are no pairs of the form (4, v)
33  ]
34 
35  The number K is called the range of the relation.
36 
37  The individual lists are guaranteed to be sorted and free of
38  duplicates.
39 
40  TODO: IntRelations, along with the efficient way of constructing
41  them in causal_graph.cc, could be useful for other parts of the
42  planner, too. If this is the case, they should be moved to a
43  different source file.
44 
45  TODO: IntRelations currently only work for relations on {0, ..., K -
46  1}^2. They could easily be generalized to relations on {0, ..., K -
47  1 } x S for arbitrary sets S. Our current code only requires that S
48  is hashable and sortable, and we have one assertion that checks that
49  S = {0, ..., K - 1}. This could easily be changed if such a
50  generalization is useful anywhere in the code.
51 */
